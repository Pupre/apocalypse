# Indoor UI Clarity Follow-up Design

- Status: approved
- Created: 2026-04-02
- Last updated: 2026-04-03
- Scope: second-pass indoor UI clarity improvements after the first reading-first restructure

## Purpose

`Indoor UI Restructure Design`로 실내 화면의 큰 구조는 정리됐지만, 실제 플레이 피드백 기준으로는 아직도 "빠르게 읽히는가" 관점에서 아쉬움이 남아 있다.

문제는 정보량 자체보다 정보가 보이는 방식이다.

- `허기`, `갈증`, `체력`, `피로`가 전부 텍스트라서 상태칩처럼 읽히지 않는다.
- `소지품`과 `장착중`은 같은 종류의 텍스트 리스트처럼 보여서 경계가 약하다.
- 구조도는 전체 오버레이로만 열 수 있어서 매번 호출해야 하고, 짧은 동선 판단에는 번거롭다.

이 문서는 실내 UI를 다시 갈아엎는 것이 아니라, 현재 reading-first 구조 위에 `시각적 계층`과 `즉시 읽힘`을 더하는 두 번째 설계를 정의한다.

## Design Role

이 설계는 다음을 다룬다.

- 상태칩 가독성 개선
- 정확한 수치 확인 방식
- 상시 미니맵과 전체 구조도 분리
- 가방 시트에서 `소지품 / 장착중`의 시각적 구분 강화

이 설계는 다음을 다루지 않는다.

- 실내 게임 규칙 변경
- 외부 UI 재설계
- 최종 아트 스타일 확정
- 신규 생존 시스템 추가

## Core Goals

1. 실내에서 현재 상태를 "읽지 않고도" 대략 파악할 수 있게 한다
2. 미니맵을 항상 참고할 수 있게 하되, 메인 독해 흐름은 망치지 않는다
3. 가방 안에서 `소지 중인 것`과 `장착 중인 것`을 집중해서 비교하지 않아도 구분되게 만든다
4. 정확한 수치는 필요할 때만 꺼내 보게 한다
5. 모바일에서도 성립하는 정보 밀도를 유지한다
6. 실내 미니맵은 항상 보이되 전체 구조를 과하게 누설하지 않는다
7. 상세 정보 패널은 닫기와 위치가 명확해야 하며 메인 행동 영역을 가리지 않는다

## Design Summary

실내 UI는 다음 세 층으로 나뉜다.

1. `항상 보이는 빠른 정보`
2. `상황 판단용 본문`
3. `필요할 때만 여는 상세 정보`

이 설계의 핵심은 `상태`, `위치`, `동선`, `가방`을 전부 같은 텍스트 레벨에 두지 않는 것이다.

## Persistent Status Chips

현재 상단바의 생존 정보는 텍스트 라벨이 아니라 `칩`처럼 보여야 한다.

### Status Chip Rules

상태칩은 아래 네 개를 유지한다.

- `허기`
- `갈증`
- `체력`
- `피로`

각 칩은 다음 요소를 가진다.

- 작은 아이콘
- 짧은 단계 텍스트
- 상태에 따른 배경 또는 톤 차이

예시:

- `허기짐`
- `목마름`
- `안정`
- `피곤`

핵심은 `허기: 허기짐`처럼 긴 문장을 계속 읽게 하지 않는 것이다. 메인 화면에서는 칩만 보고도 현재 상태가 대략 들어와야 한다.

### Status Detail Panel

정확한 수치는 메인 화면이 아니라 상태칩 상세 패널에서만 보여준다.

칩을 누르면 작은 상세 패널 또는 하단 시트가 열리고, 다음을 보여준다.

- 현재 수치 예: `62 / 100`
- 현재 단계 예: `보통`
- 간단한 설명 예: `갈증이 0이 되면 체력이 계속 감소한다`
- 관련 회복 수단 예: `물 / 음료`
- 명시적인 `닫기` 버튼

세부 표기 규칙:

- 정확한 수치는 소수점 없이 정수로만 표시한다
- 메인 화면은 계속 단계형을 유지한다

즉:

- 메인 화면 = 단계형
- 상세 화면 = 정확한 수치 + 짧은 규칙 설명

## Minimap Split

현재 실내 구조도는 오버레이로만 열 수 있다. 이 방식은 전체 구조 확인에는 좋지만, 매 이동 때마다 누르기엔 번거롭다.

따라서 구조도는 두 층으로 나눈다.

### Mini Minimap

실내 메인 화면에는 작은 상시 미니맵이 항상 보인다.

미니맵 규칙은 기존과 같다.

- 현재 위치 표시
- 방문한 구역 표시
- 현재 위치와 인접한 미확인 구역은 `?`
- 잠긴 길은 잠김 표시

중요한 점:

- 전체 구조를 한 번에 다 보여주지 않는다
- 현재 동선 판단에 필요한 만큼만 항상 보여준다

### Full Map Overlay

기존 `구조도` 버튼은 유지하되, 의미를 더 명확히 한다.

- 메인 화면의 작은 미니맵 = 현재 동선용
- `구조도` 버튼 = 더 크게 보는 전체 구조도 오버레이

필요하면 버튼 라벨은 `전체 구조도`처럼 더 명확하게 바꿀 수 있다.

### Layout Rule

작은 미니맵은 메인 화면에서 항상 보이되, 메인 읽기 흐름을 깨지 않아야 한다.

우선순위는 다음과 같다.

1. 현재 위치 설명
2. 행동 선택
3. 상시 미니맵

그래서 미니맵은 크지 않아야 하며, 본문을 밀어내는 주인공이 되면 안 된다.

레이아웃 기준:

- 넓은 화면: 상단바 아래 우측 또는 본문 우상단의 작은 카드
- 좁은 화면: 상단바 아래 별도 얇은 카드로 접지 않고 배치

즉 "사라지지 않는 작은 지도"는 유지하되, 메인 텍스트보다 커지지 않게 한다.

### Discovery Rule Refinement

상시 미니맵은 `현재 위치 중심`으로만 보여야 한다.

- 현재 위치는 항상 중심 근처에 둔다
- 현재 위치에서 이어진 구역만 즉시 드러난다
- 이미 방문한 구역은 유지하되, 멀리 떨어진 구역이 미니맵의 주인공이 되면 안 된다
- 전체 연결 관계 확인은 여전히 `전체 구조도` 오버레이의 역할이다

즉:

- 상시 미니맵 = 지금 주변 판단
- 전체 구조도 = 건물 전체 파악

## Inventory Sheet Clarity

가방 시트는 구조 자체는 맞지만, 현재는 `소지품`과 `장착중`의 시각적 차이가 약하다.
또한 현재 아이템 상세가 별도 겹침 패널로 떠서 본문과 충돌한다. 이번 설계에서는 가방 시트 내부에 `목록 + 상세`를 함께 배치한다.

### Bag Tabs

`소지품 / 장착중`은 단순 텍스트 버튼이 아니라 `세그먼트 탭`처럼 보여야 한다.

필수 규칙:

- 현재 선택된 탭은 배경/경계/톤이 분명히 다르다
- 선택되지 않은 탭은 눌러야 할 버튼처럼 보인다
- 두 탭이 같은 종류의 텍스트처럼 보여선 안 된다
- `장착중` 탭은 현재보다 더 강한 선택 강조를 가진다

### Bag Layout

가방 시트는 내부를 두 영역으로 나눈다.

- 왼쪽: `소지품` 또는 `장착중` 목록
- 오른쪽: 선택한 항목의 상세 정보

이 규칙의 목적은 두 가지다.

- 상세 패널이 메인 화면을 덮지 않게 한다
- 어떤 아이템을 보고 있는지 시선이 자연스럽게 이어지게 한다

즉 아이템 상세는 별도 떠다니는 팝업보다 `가방 내부 우측 상세 패널`로 고정한다.

### Inventory Rows

소지품 목록의 각 행도 단순 텍스트가 아니라 `선택 가능한 아이템 행`처럼 보여야 한다.

필수 요소:

- 누를 수 있는 행이라는 시각 신호
- 선택 시 강조 상태
- 수량 정보

예:

- `에너지바 x1`
- `작은 배낭 x1`

### Equipped Rows

장착중 목록은 "버튼 리스트"보다 "현재 상태 목록"처럼 보이는 쪽이 낫다.

예:

- `등 · 작은 배낭`
- `손 · 작업 장갑`

즉:

- `소지품`은 상호작용 목록
- `장착중`은 현재 상태 목록

같은 컴포넌트처럼 보이면 안 된다.

추가 규칙:

- `장착중` 목록은 눌러서 상세를 볼 수 있더라도, 기본 인상은 버튼보다 상태 카드에 가깝게 만든다
- 슬롯명, 아이템명, 핵심 효과를 한 눈에 구분할 수 있어야 한다
- 빈 슬롯도 `비어 있음` 상태로 정돈되게 보여야 한다

## Icon Use

이번 단계부터는 `텍스트만으로 구분`하려는 방식을 줄여야 한다.

아이콘은 다음 용도에 우선 사용한다.

- 생존 수치 칩
- 가방 탭 또는 장착 슬롯
- 필요하면 미니맵/전체 구조도 버튼

단, 아이콘이 의미를 숨기면 안 된다. 텍스트를 완전히 없애는 것이 아니라, 빠른 분류를 돕는 역할이어야 한다.

## Interaction Hierarchy

실내 메인 화면에서 플레이어의 시선 흐름은 아래 순서가 되어야 한다.

1. 현재 위치
2. 현재 상태칩
3. 현재 위치 설명
4. 행동 선택
5. 작은 미니맵
6. 가방/상세 정보

현재처럼 가방, 장착, 구조도, 상태가 한 레벨로 섞이면 안 된다.

## Location Strip

현재 위치 텍스트는 메인 행동 영역과 겹치지 않게, 상단바 아래의 별도 줄로 유지한다.

이 줄은 드롭다운처럼 동작할 필요는 없다. 현재 단계에서는 `항상 보이는 위치 스트립`이 더 직관적이다.

목표는 다음과 같다.

- 위치가 행동 카드와 같은 레벨로 섞이지 않게 한다
- 플레이어가 지금 어디에 있는지 항상 빠르게 확인하게 한다
- 위치가 바뀌면 설명과 행동이 함께 자연스럽게 바뀌도록 한다

## Mobile-First Constraint

이 설계는 모바일을 전제로 한다.

따라서:

- 상시 미니맵은 작아야 한다
- 상태칩은 길지 않아야 한다
- 탭과 버튼은 손가락으로 누르기 쉬워야 한다
- 상세 정보는 별도 시트/패널로 빼야 한다

즉 "한 화면에 다 보여주기"보다 "자주 필요한 것만 상시 표시하고, 나머지는 한 번 탭해서 열기"가 기본 규칙이다.

## Compatibility with Current System

이번 설계는 기존 실내 시스템과 충돌하지 않는다.

- `IndoorDirector`가 가진 상태/인벤토리/맵 정보는 그대로 쓴다
- 발견 규칙, 잠긴 길 규칙, 아이템 액션 규칙은 그대로 유지한다
- 바뀌는 것은 어디에, 어떤 밀도로, 어떤 시각 신호로 보여주느냐뿐이다

## Success Criteria

이 설계가 잘 반영되면 플레이어는 다음을 더 빠르게 할 수 있어야 한다.

- 현재 상태를 한눈에 읽는다
- 짧은 동선은 작은 미니맵만 보고 판단한다
- 전체 구조가 궁금할 때만 큰 구조도를 연다
- 가방에서 `소지품`과 `장착중`을 헷갈리지 않는다
- 정확한 수치는 필요할 때만 상세 패널에서 확인한다
- 상태 상세 패널은 명시적으로 닫을 수 있다
- 정확한 생존 수치는 정수로만 보여도 충분히 읽힌다
- 아이템 상세는 메인 화면을 덮지 않고 가방 내부 우측에서 안정적으로 읽힌다

## Out of Scope

이번 설계는 다음을 포함하지 않는다.

- 외부 미니맵/UI 변경
- 상태칩 애니메이션
- 최종 픽셀아트 UI 스킨
- 장비 시스템 규칙 변경
- 인벤토리 용량 규칙 재설계
