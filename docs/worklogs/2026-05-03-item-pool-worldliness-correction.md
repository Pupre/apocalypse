# 2026-05-03 아이템 풀 생활감 보정

## 배경

첫 500개 아이템 확장안은 숫자는 맞췄지만 생존 물자에 너무 집중되어 있었다. 예를 들어 곡물바, 육포, 비상 비스킷처럼 비슷한 물건을 상태나 수식어만 바꿔 늘리는 느낌이 강했다. 이 방향은 파밍 기대감을 만들기보다 “데이터를 불렸다”는 인상을 줄 수 있어서 바로 수정했다.

## 수정 방향

- 모든 물건이 생존을 위해 존재하지는 않는다는 전제를 반영했다.
- 파밍지는 재난용 창고가 아니라 사람들이 살고 일하던 장소이므로, 생활 흔적과 사적인 물건이 함께 나와야 한다.
- 직접 성능이 낮은 물건도 거래품, 단서, 감정적 선택, 조합 재료가 될 수 있도록 태그를 부여했다.

## 구현

- `tools/generate_survival_item_expansion.py`의 생성 규칙을 재작성했다.
- 500개 아이템 구성을 다음처럼 재배치했다.
  - 음식 45개
  - 음료 18개
  - 의료/위생 28개
  - 도구 35개
  - 생활 재료 70개
  - 수납/가방 25개
  - 착용 장비 70개
  - 문서/단서 32개
  - 사적 물건 47개
  - 전자기기/부품 30개
  - 가정용품 40개
  - 임시 제작품 60개
- `ordinary_world`, `personal`, `electronics`, `household`, `story`, `barter` 태그를 추가해 생활감 있는 물건을 시스템적으로 구분할 수 있게 했다.
- 조합식은 결과물 이름을 기준으로 방수, 보온, 불, 의료, 수납, 수리, 신호, 은닉/문서 계열 재료를 고르게 붙이도록 다시 생성했다.
- 기존보다 명백한 수식어 반복형 아이템을 줄이고, 아이템명 중복을 테스트로 막았다.
- 확장 루트 후보의 기본 가중치를 더 낮춰, 새 아이템이 기존 건물의 대표 물건을 밀어내지 않고 “가끔 섞이는 발견”처럼 작동하게 했다.

## 검증 관점

- `res://tests/unit/test_survival_item_expansion.gd`에 아이템 풀 성격 검증을 추가했다.
- 확장 아이템 500개 중 대부분이 `ordinary_world` 태그를 가져야 한다.
- 사적 물건, 문서, 전자기기, 가정용품처럼 직접 생존 성능이 낮아도 세계를 설명하는 물건이 충분히 포함되어야 한다.
- “밀봉/고열량/작은 곡물바”처럼 숫자를 늘리기 위한 반복 변주가 다시 늘어나지 않도록 제한했다.
