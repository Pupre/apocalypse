# 2026-05-04 대지역 이동 사건 1차

## 목표

- 12x12 야외 맵의 대지역이 단순히 이름과 배치만 다른 구역으로 보이지 않도록, 실제 이동 중 체감되는 압박을 추가한다.
- 사용자가 말한 “탐험을 하는 느낌”을 위해 건물 안 선택지뿐 아니라 건물 사이를 지나가는 순간에도 지역 성격이 드러나게 한다.

## 변경 사항

- `game/scripts/outdoor/outdoor_controller.gd`에 `DISTRICT_TRAVEL_INCIDENTS`를 추가했다.
  - 각 대지역은 한 번씩만 발동하는 이동 사건을 가진다.
  - 사건은 기존 `apply_outdoor_hazard_contact()` 흐름을 사용해 체온 손실, 피로 증가, 소량 부상, 이동 시간 지연을 적용한다.
  - 기존 야외 위험 HUD와 서리 플래시를 재사용해 새 UI를 만들지 않고도 즉시 피드백을 준다.
- `bind_run_state()`로 처음 야외에 배치될 때는 사건을 발동하지 않도록 했다.
  - 플레이 시작이나 테스트 초기화 순간에 갑자기 피해를 입지 않는다.
  - 실제로 다른 대지역으로 넘어갈 때만 이동 사건이 발동한다.
- `game/tests/unit/test_outdoor_controller.gd`에 검증을 추가했다.
  - 국도 검문·휴게 구역에서 남부 물류 벨트로 넘어갈 때 지역 사건이 발동하는지 확인한다.
  - 체온, 피로, 시간 비용이 실제로 바뀌는지 확인한다.
  - 같은 지역 사건이 한 야외 세션에서 반복 처벌되지 않는지 확인한다.

## 설계 판단

- 이번 단계에서는 별도 컷신이나 신규 이미지를 만들지 않았다.
  - 이 기능은 빠르게 반복되는 야외 이동 피드백이므로, 전체화면 일러스트보다 HUD와 화면 플래시가 더 적합하다고 판단했다.
  - 이후 특정 대지역 첫 진입이나 큰 발견에는 별도 일러스트 연출을 붙일 수 있다.
- 사건 수치는 위험 타일보다 약하게 두었다.
  - 목적은 “이 구역은 지나가기만 해도 성격이 있다”는 리듬을 만드는 것이지, 이동 자체를 과도하게 벌주는 것이 아니기 때문이다.
