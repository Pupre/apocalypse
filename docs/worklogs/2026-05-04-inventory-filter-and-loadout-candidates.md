# 2026-05-04 가방 필터와 장착 후보 UX 확장

## 목적

- 아이템 풀이 커질수록 가방 목록에서 원하는 물건을 찾는 시간이 늘어난다.
- 장착 탭을 분리했지만, 장착 가능한 물건을 찾으려면 여전히 가방 탭과 장착 탭을 오가야 했다.
- 이번 패스의 목표는 "가방은 빠르게 좁혀 보고, 장착 탭은 바로 갈아끼우는 곳"으로 역할을 더 분명히 하는 것이다.

## 구현

- 가방 탭에 빠른 필터 레일을 추가했다.
  - `전체`
  - `생존`
  - `도구`
  - `장착`
  - `읽을 것`
  - `무거움`
- 필터는 일반 가방 탐색 중에만 노출된다.
  - 조합 모드에서는 기존처럼 `기준 재료`, `조합 가능`, `다른 물건` 구조가 우선한다.
- 장착 탭에 `장착 후보` 영역을 추가했다.
  - 인벤토리 안의 `equip_slot` 보유 아이템을 자동으로 모은다.
  - 비어 있는 슬롯 후보는 `장착`, 이미 같은 슬롯에 장비가 있으면 `교체` 버튼을 보여 준다.
  - 후보는 장착 슬롯 순서와 아이템 이름 순서로 정렬한다.
- 아이템 상세 행동의 우선순위를 바꿨다.
  - 소비, 사용, 장착, 읽기 같은 즉시 필요한 행동은 주요 행동 영역에 둔다.
  - `버린다`는 실수 터치를 줄이기 위해 보조 행동 영역으로 내렸다.
- AI 이미지 생성으로 필터 전용 레일 `resources/ui/master/sheet/inventory_filter_rail.png`를 만들었다.
  - 텍스트는 이미지에 굽지 않고, Godot 버튼 텍스트가 올라가는 배경으로만 사용한다.
  - `UiKitResolver`와 마스터 매니페스트에 연결했다.
