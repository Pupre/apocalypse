# 2026-05-03 야외 지구 지도 피드백

## 목적

- 야외 맵이 12x12로 커진 뒤, 전체 지도에서 각 지역의 성격 차이가 충분히 드러나지 않는 문제가 있었다.
- 사용자는 멀리 이동할 때 "다음에는 어떤 동네로 들어가는가"를 빠르게 읽을 수 있어야 하므로, 방문한 지구의 색 차이를 전체 지도에 먼저 붙였다.

## 구현

- `OutdoorMapView.build_snapshot()`에 `district_blocks`를 추가했다.
  - 방문한 블록만 `district_id`, `layout_id`, 블록 좌표, 월드 사각형을 노출한다.
  - 미탐색 블록은 기존 `fog_blocks` 뒤에 남겨, 지도에서 지구 정보를 미리 누설하지 않는다.
- 전체 지도 렌더링에서 방문 블록 단색 채우기를 지구별 색면으로 교체했다.
  - 북부 시장가, 동부 의료지구, 남부 주거지, 남동 공업지대, 서부 대피선, 중앙 환승로, 외곽 혼합지대가 서로 다른 낮은 채도의 색을 가진다.
  - 도로, 장애물, 건물, 플레이어 마커는 기존 순서대로 위에 그려져 기능적 가독성을 유지한다.
- `test_outdoor_map_view.gd`에 지구 스냅샷 계약 검증을 추가했다.
  - 방문한 블록은 지구 ID를 노출한다.
  - 안개 뒤의 미탐색 블록은 지구 ID를 노출하지 않는다.

## 검증

- 샌드박스 안의 Godot headless 실행은 Windows 세션에서 프로젝트 시작/종료 중 타임아웃과 크래시가 발생했다.
- 같은 명령을 샌드박스 밖에서 실행하면 정상 통과했다.
- 통과한 테스트:
  - `res://tests/unit/test_outdoor_map_view.gd`
  - `res://tests/unit/test_outdoor_controller.gd`
  - `res://tests/unit/test_outdoor_map_expansion.gd`
  - `res://tests/smoke/test_first_playable_loop.gd`

## 다음 작업

- 전체 지도에 지구 범례와 핵심 파밍지 후보를 더 붙이면, 넓어진 맵을 단순 이동 공간이 아니라 장기 탐험 계획 도구로 쓸 수 있다.
- 지구 색 정보는 이후 건물별 스토리 컷신, 대형 선택지, 위험 구역 안내와 연결하기 좋은 기반이다.
