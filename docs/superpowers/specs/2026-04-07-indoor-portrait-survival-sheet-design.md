## 목적

이 문서는 현재 가로 지향 실내 UI를 `세로 전용 모바일 게임` 문법으로 재구성하기 위한 1차 설계다.

이번 라운드의 핵심 목표는 세 가지다.

- 실내 메인 플레이를 `세로 스택 + 바텀시트` 중심으로 바꾼다.
- `가방 / 조합 / 도감`을 따로따로 뜨는 창이 아니라 하나의 `SurvivalSheet` 흐름으로 합친다.
- 조합을 `슬롯 A / 슬롯 B` 같은 도구창 문법이 아니라 `아이템을 읽고 -> 이걸로 조합 -> 호환 재료를 고른 뒤 -> 조합`하는 모바일 문법으로 바꾼다.

이번 설계는 화면 비율만 바꾸는 작업이 아니다.

- 기존 생존 시스템, 인벤토리, 조합 데이터, codex, 실내 기억, 드롭 영속화는 유지한다.
- 대신 플레이어가 정보를 읽고 행동을 고르는 `표현 계층`과 `상호작용 문법`을 세로 전용으로 다시 세운다.

## 현재 진단

현재 구현은 시스템적으로는 확장되어 있지만, 화면 문법은 여전히 가로 게임 습관이 강하다.

### 1. 실내 메인 화면이 가로 분할 사고를 갖고 있다

- 상단 정보, 본문 카드, 미니맵, 행동 리스트, 가방 시트가 서로 다른 패널로 나뉘어 있다.
- `ContextRow`, `BagSheet/ContentRow` 같은 옆으로 벌어지는 배치가 많다.
- 정보가 많기보다 `동시에 주인공이 되는 블록`이 많아서 난잡하게 느껴진다.

### 2. 가방과 조합이 흐름상 분리되어 있다

- 지금은 조합을 별도 시트로 열고, 결과를 만든 뒤 설명을 보려면 다시 가방으로 가야 한다.
- 플레이어 입장에서는 “조합을 했다”와 “그 결과물을 이해했다”가 같은 행동 흐름이어야 하는데, 현재는 그 사이에 창 전환이 끼어든다.

### 3. 현재 조합 문법은 모바일 감각과 잘 맞지 않는다

- `재료 A / 재료 B` 슬롯은 기능적으로는 단순하지만, 실제 감각은 `가방을 뒤지는 생존자`보다 `툴에 재료를 꽂는 사용자`에 가깝다.
- 세로 모바일에서는 이 문법이 특히 도구창처럼 느껴진다.

## 목표 경험

플레이어는 실내에서 다음 경험을 받아야 한다.

1. 실내 메인 화면을 열면 정보가 세로로 차분하게 쌓여 있고, 무엇을 먼저 봐야 할지 바로 이해된다.
2. 가방을 열면 하단 바텀시트에서 아이템 설명과 행동을 읽을 수 있다.
3. 어떤 아이템이 조합 재료로 흥미로워 보이면 `이걸로 조합`을 눌러 같은 가방 안에서 실험을 이어갈 수 있다.
4. 조합 모드에 들어가도 가방 목록은 그대로 유지된다.
5. 호환되는 두 번째 재료만 목록 안에서 강하게 강조되어, 조합 가능한 걸 즉시 파악할 수 있다.
6. 두 번째 재료를 눌렀을 때 결과를 미리 스포일러하지 않고, 설명을 읽은 뒤 `조합` 버튼을 누르게 한다.
7. 조합 성공 직후 결과물이 같은 바텀시트에 즉시 열려, 방금 만든 아이템을 바로 이해할 수 있다.

## 범위

이번 설계는 아래를 포함한다.

- 세로 전용 실내 메인 레이아웃
- 실내 `BagSheet`, `CraftButton`, `CodexButton` 흐름 재구성
- 가방/조합/도감을 통합한 `SurvivalSheet` 바텀시트
- `이걸로 조합` 중심의 모바일 조합 흐름
- 조합 가능 재료의 강한 하이라이트
- 조합 성공 직후 결과물 즉시 상세 표시

이번 설계에서 의도적으로 제외하는 것:

- 실외 카메라와 실외 월드의 세로 전환
- 실외 HUD 재설계
- 전역 기준 해상도와 전체 앱 프레임 재설정
- 따뜻한 수프 등 수치 밸런스 재조정
- 전면적 아트 리디자인

즉 이번 라운드는 `실내 문법을 세로 전용으로 고정하는 1차 공사`다.

## 선택된 접근

이번 설계는 다음 접근을 선택한다.

- 실내에서 먼저 `세로 단일 컬럼 + 통합 바텀시트`를 완성한다.

이 접근을 선택한 이유:

- 실내는 텍스트 읽기, 행동 선택, 인벤토리 확인, 조합 시도가 밀접하게 엮여 있어 세로 문법의 이점을 가장 크게 받는다.
- 실내에서 바텀시트 패턴을 먼저 고정해야, 이후 실외 가방/도감에도 같은 문법을 일관되게 가져갈 수 있다.
- 실외까지 한 번에 손대면 카메라와 월드 레이아웃까지 동시에 흔들려 리스크가 커진다.

## 실내 메인 레이아웃

실내 메인 화면은 `하나의 세로 스택`으로 정리한다.

- 상단 압축 상태줄
- 위치 / 요약 카드
- 결과 피드백
- 행동 리스트
- 필요 시 미니맵 진입 버튼

핵심 원칙은 아래와 같다.

- 같은 깊이의 큰 패널을 좌우로 나누지 않는다.
- 실내 메인 화면에서 항상 읽어야 하는 정보는 세로 한 흐름으로 정렬한다.
- `가방 / 조합 / 도감`은 메인 화면에 동시에 떠 있지 않고, 바텀시트로 진입한다.

즉 메인 화면은 `현재 상황 파악 + 행동 선택`에 집중하고, 아이템 관리 계층은 바텀시트로 내린다.

## SurvivalSheet 구조

`SurvivalSheet`는 실내 하단에서 올라오는 하나의 통합 바텀시트다.

이 시트는 별도 독립 창 3개를 대체한다.

- 기존 `BagSheet`
- 기존 공용 `CraftingSheet`
- 기존 별도 `CodexPanel`

이번 1차에서 `SurvivalSheet`는 아래 탭/모드를 가진다.

- `가방`
- `도감`

조합은 별도 탭이 아니다.

- 조합은 `가방` 탭 안의 `조합 모드`로 동작한다.
- 즉 조합은 인벤토리와 분리된 도구창이 아니라, 인벤토리 문법의 연장이어야 한다.

## 가방 탭 문법

가방 탭의 주 구조는 다음과 같다.

- 상단: 현재 탭 헤더와 상태 문구
- 본문: 스크롤 가능한 아이템 목록
- 하단: 선택 아이템 상세 바텀시트

아이템을 탭하면 바텀시트가 그 아이템을 설명한다.

표시 정보:

- 이름
- 짧은 분류 / 태그
- 설명
- 효과 / 사용 힌트
- 가능한 행동 버튼

이때 버튼은 현재 아이템 상태에 따라 바뀐다.

예:

- `사용`
- `장착`
- `읽는다`
- `버린다`
- `이걸로 조합`

## 조합 모드 문법

### 진입

플레이어가 가방 탭에서 특정 아이템을 선택하고 `이걸로 조합`을 누르면 조합 모드에 들어간다.

조합 모드 진입 시:

- 첫 재료를 `기준 재료`로 고정한다.
- 바텀시트 상단에 작은 칩으로 표시한다.
- 가방 목록은 그대로 유지한다.
- 호환되는 두 번째 재료만 목록 안에서 강하게 하이라이트한다.

### 조합 모드 중 목록 동작

조합 모드에서도 플레이어는 여전히 가방 목록을 자유롭게 탭할 수 있다.

- 호환 재료를 탭하면 그 아이템 설명이 바텀시트에 열린다.
- 비호환 재료를 탭해도 조합 모드는 자동 해제되지 않는다.
- 대신 상태 문구로 `지금 기준 재료와는 맞지 않는다`를 명확히 보여준다.

이 결정을 선택한 이유:

- 잘못 눌렀다고 조합 모드가 풀리면 모바일에서 실수 비용이 너무 커진다.
- 조합 모드는 `도전 상태`로 유지하고, 플레이어가 취소를 명시적으로 누를 때만 빠지게 하는 편이 더 안정적이다.

### 조합 확정

두 번째 재료를 탭했다고 즉시 조합되지는 않는다.

- 결과 미리보기는 띄우지 않는다.
- 바텀시트의 주 행동 버튼만 `조합`으로 바뀐다.
- 플레이어는 설명을 읽은 뒤 `조합`을 누른다.

즉 흐름은 아래와 같다.

1. 아이템을 본다
2. `이걸로 조합`
3. 하이라이트된 다른 아이템을 탭한다
4. 설명을 읽는다
5. `조합`

## 조합 결과 처리

조합 성공 시:

- 현재 재료 소모 / 도구 조건 / charge 소모 로직은 기존 `RunState.attempt_craft(...)`를 유지한다.
- 조합 모드는 종료된다.
- 결과물 아이템이 같은 바텀시트에 즉시 선택 상태로 열린다.

이 동작은 중요하다.

- 조합 시트 -> 가방 시트로 다시 이동하지 않는다.
- “방금 만든 아이템이 무엇인지”를 같은 맥락에서 바로 읽게 만든다.

조합 실패 시:

- 실패 메시지는 바텀시트 상태 문구와 실내 결과 피드백에 동시에 반영한다.
- 예: `라이터가 필요하다`
- 예: `실내에서만 가능한 조합이다`

## 도감 탭 문법

도감은 `SurvivalSheet` 안의 두 번째 탭으로 들어간다.

도감은 기존 설계 원칙을 유지한다.

- 전체 레시피 목록 기반
- 미발견 레시피는 `???`
- 발견 레시피는 재료, 결과, 설명, 조건 표시

다만 표현 방식은 세로 문법에 맞춘다.

- 카테고리 헤더를 세로 목록 안에 배치한다.
- 각 항목은 카드가 아니라 `압축된 세로 행`으로 표시한다.
- 이미 알고 있는 레시피를 빠르게 다시 찾는 데 초점을 둔다.

## 하이라이트 규칙

현재 하이라이트는 존재를 놓치기 쉬운 수준으로 약하다.

1차 실내 전환에서는 하이라이트를 더 공격적으로 바꾼다.

- 테두리만 바꾸는 수준이 아니라 배경색, 밝기, 펄스를 함께 사용한다.
- Easy에서는 항상 켠다.
- Hard에서는 기존처럼 꺼진다.

하이라이트의 목적은 `예쁜 효과`가 아니라 `놓치지 않게 하는 기능적 강조`다.

## 상태 기계

실내 바텀시트는 아래 상태를 가진다.

- `hidden`
  - 시트 닫힘
- `inventory_browse`
  - 일반 가방 탐색
- `inventory_craft_select`
  - 기준 재료가 고정된 조합 모드
- `codex_browse`
  - 도감 탐색

이 상태 기계를 분리하는 이유:

- 지금처럼 가방, 조합, 도감이 각각 따로 열린 상태를 제어하지 않아도 된다.
- 한 시트 안에서 무엇이 가능한지 분명해진다.

## 런타임 영향

이번 설계는 데이터 구조를 크게 바꾸지 않는다.

- 인벤토리 아이템 데이터 구조 유지
- codex 지식 구조 유지
- lighter charge / 도구 조건 구조 유지
- `RunState.attempt_craft(...)` 유지

주된 변경점은 UI 계층과 시트 상태 관리다.

예상되는 코드 변화 축:

- `indoor_mode.tscn`
- `indoor_mode.gd`
- 공용 조합 시트 대체용 새 `survival_sheet` 씬/스크립트
- `run_shell.tscn`의 sheet 연결 구조
- indoor 관련 테스트

## 테스트 전략

이번 라운드에서 최소한 아래 검증이 필요하다.

1. 실내 화면이 세로형 레이아웃으로 뜬다.
2. 가방을 열면 아이템 상세가 바텀시트에 표시된다.
3. `이걸로 조합` 진입 시 기준 재료가 고정되고 호환 재료가 강조된다.
4. 비호환 아이템 탭 시 조합 모드는 유지된다.
5. 호환 아이템 탭 후 `조합` 버튼이 활성화된다.
6. 조합 성공 후 결과물이 바텀시트에 즉시 열린다.
7. 도감 탭 진입과 discovered / undiscovered 렌더링이 유지된다.
8. easy / hard에 따른 하이라이트 on/off가 유지된다.

## 이후 단계

실내 1차가 안정화되면 다음 라운드에서 다룰 항목은 아래다.

- 실외 HUD의 세로 전환
- 세로 카메라에 맞춘 연속 스크롤형 실외 월드 구도
- 플레이어 하단 오프셋 카메라
- 실외 가방/도감도 같은 `SurvivalSheet` 문법으로 통합

즉 이번 설계는 최종 목적지가 아니라, `세로 전용 전체 게임`으로 가기 위한 첫 번째 구조 전환이다.
